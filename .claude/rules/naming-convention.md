# Naming Convention Rules

> All output files generated by agents MUST follow these naming conventions.

## General Pattern

```
[UC-ID]_[feature-name]_[type]_[date-or-version]_v[N].[ext]
```

### Components

| Component        | Format              | Example            | Notes                                    |
| :--------------- | :------------------ | :----------------- | :--------------------------------------- |
| `UC-ID`          | `UC-XXX`            | `UC-101`           | Use Case ID from requirement document    |
| `feature-name`   | `lowercase-kebab`   | `user-login`       | Short descriptive name, max 3-4 words    |
| `type`           | See table below     | `extracted`        | Indicates the output type                |
| `date`           | `YYYYMMDD`          | `20260324`         | Date the file was created                |
| `version`        | `vN`                | `v1`               | Version number for finalized files       |

## Output Types

| Type              | Suffix Value          | Extension | Skill              |
| :---------------- | :-------------------- | :-------- | :----------------- |
| Audited Req       | `audited`             | `.md  `   | qc-uc-review-report       |
| Scenario          | `scenarios`           | `.md`     | qc-scenario-design-report |
| Test Case         | `testcases`           | `.xlsx`   | qc-tc-design-report       |
| Extracted Req     | `extracted`           | `.md`     | Any agent          |
| Questions         | `questions`           | `.md`     | qc-qna       |

## Version Control
- Files use `v[N]` version suffix — they are **immutable** once created.
- When creating a new version, increment `N` (v1 → v2 → v3).
- Never delete or overwrite a version.
